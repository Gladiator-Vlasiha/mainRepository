Используемый стэк:
Средства разработки: Java

Cреда разработки: IntelliJ IDEA
JDK: 17
Cистема автоматической сборки : Gradle 7.
База данных:  MongoDB 5
Протокол: HTTP, порт 80
Контейнер сервлетов: Jetty 11.0.9
Плагин запуска контейнеров: Gretty 4.0.2
Библиотека логирования: Log4j
Сторонняя библиотека JSON.simple


При запуске из среды IDE:

	plugin gretty: 
			appRun

Вэб интерфейс для добавления платежей в БД
	http://localhost:8080/OvermobileProject/static/form.html

Путь для прямых POST запросов(POSTMAN,APITester)
	http://localhost:8080/OvermobileProject/main

Для добавления из консоли/командной строки
	команда ADD_PAY 
	и далее через пробел 3 значения id userID sum
	example "ADD_PAY 1 23764 999"

Для работы в cmd выполнить в каталоге с файлом команду:
	OvermobileProject>java -jar OvermobileProject-1.0-SNAPSHOT.jar


Описание сути программы:
*servlet принимает post-запросы с информацией о платежах,
 в теле каждого ожидается информация об одном платеже: 
json-сообщение с полями id (строка), userId (число) и sum (число).
*сервер синхронно отвечает на каждый такой запрос 
сообщением с единственным полем status (число):
 0, если это новый платёж (информации о платеже с таким id
 ранее не поступала) или 1, если такой платёж уже был.
*сервер сохраняет информацию о каждом новом платеже в 
отдельном документе в хранилище; при получении запроса о
 платеже, который уже был - новой записи не делает.
